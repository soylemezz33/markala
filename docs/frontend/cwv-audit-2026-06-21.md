## 2. Bulgular

### LCP — Yüksek öncelik
- **Hero `<h1>` (LCP adayı) framer-motion ile `initial={{ opacity:0 }}`**
  (`components/home/hero-carousel.tsx`). SSR HTML başlığı opacity:0 inline stille
  basıyor; metin ancak JS inip hydration + animasyon (0.15s+ delay) bitince
  görünür oluyor. LCP, HTML boyamasına değil tüm JS hattına bağlanıyor.
  → **Bu PR'da düzeltildi** (bkz. §3).
- **Hero `<Image unoptimized>`** (`hero-carousel.tsx`) ve sitede toplam **24
  `unoptimized` next/image** kullanımı. Sebep: ürün görselleri `/api/mockup`
  SVG döndürüyor, optimizer SVG'yi işlemiyor. Ancak picsum/unsplash/pexels
  (raster) kaynaklarda `unoptimized` gereksiz → AVIF/WebP kaçırılıyor.
  → Sistemik; ürün kararı gerekiyor (bkz. Takip).

### INP / TBT — Orta öncelik
- **188 kB shared JS her sayfada**, anasayfada 320 kB. Bunun büyük kısmı
  layout'ta global mount edilen client component'lerden (`SiteHeader`,
  `CartDrawer`, `FloatingActions`, `CookieConsent`, `CtaBanner`) gelen
  framer-motion. framer-motion shared chunk'a girdiği için her rotada yükleniyor.
  → Yapısal; güvenli kademeli refactor önerisi Takip'te.

### CLS — Düşük (sorun yok)
- next/image kullanımlarının tamamı `fill` (boyutlu parent) veya açık
  `width/height` ile → intrinsic boyut biliniyor.
- Geç mount olan banner'lar (`cookie-consent`, `floating-actions`) `position:
  fixed` → normal akış dışı, reflow/CLS üretmiyor. `promo-banner` açık
  `width=1280 height=360`.
- Font: `next/font` DM_Sans `display:swap` → FOIT yok.

### Üçüncü taraf script — İyi
- `components/analytics.tsx`: GA4/GTM/Clarity/Hotjar/Meta hepsi `next/script`
  `afterInteractive` + Consent Mode v2 gate. İlk render'ı bloklamıyor.

## 3. Bu PR'daki Düzeltmeler

1. **Hero LCP düzeltmesi** (`hero-carousel.tsx`): `firstSlideRef` ile ilk slide
   ilk boyamada enter-animasyonsuz (`initial={false}`) render edilir → H1/CTA/
   görsel doğrudan `animate` (opacity:1) durumunda, SSR HTML'de görünür, LCP JS
   hattını beklemez. İlk geçişten (autoplay/nav) sonra animasyonlar normal
   çalışır. Görsel tasarım korunur (yalnızca ilk açılış statik gelir — içerik
   öncelikli, daha profesyonel his).
2. **`optimizePackageImports`** (`next.config.mjs`): phosphor-icons + framer-motion.
   Üretim bundle'ı değişmedi (ölçüldü); dev derleme hızı için eklendi.
3. **`output` env toggle** (`next.config.mjs`): `NEXT_DISABLE_STANDALONE=1` ile
   standalone kapatılabilir — Windows local'de trace-copy symlink (EPERM) hatasını
   aşar; CI/Docker'da env set edilmez, standalone aktif kalır.

**Doğrulama:** `pnpm build` ✓ (178/178 statik sayfa), `tsc --noEmit` ✓ temiz,
First Load JS regresyon yok (320 kB sabit; hero değişikliği render-timing, bundle
etkisi yok).
